# QuickLauncher 原生版
一个专注于低后台占用的原生 Win32 快捷启动器。
## 快速开始
build.bat
## 功能
- 支持自定义全局热键呼出或隐藏，默认是 `Alt+Space`
- 紧凑窗口布局，只保留搜索和结果区域
- 后台搜索带防抖，在大索引下输入更平滑
- 搜索使用分词前缀候选桶和上次匹配结果收窄，输入越长，检索越便宜，不会每次全量重扫
- 预扫描固定磁盘并缓存可启动文件
- 支持搜索并直接打开文件夹
- 支持搜索 `.exe`、`.bat`、`.cmd`、`.ps1`、`.vbs`、`.lnk`、`.msc` 等启动目标
- 支持模糊匹配和首字母匹配，适合缩写和拼音首字母搜索
- 常用结果和查询历史会提升排序
- 可以用 `/.bat:` 或 `/.lnk:` 这样的前缀优先显示对应后缀，例如 `/.bat: deploy`
- 可以用 `/dir:` 前缀优先显示目录，并浏览当前路径层级，例如 `/dir:E:\\`、`/dir:E:\\Tools` 或 `/dir:\"E:\\Program Files\" obsidian`
- `.lnk`、`.exe`、`.bat`、`.cmd` 这类可启动结果会排在文件夹之前
- 空查询会使用缓存首页列表，热键呼出时不用重新全量排序也能显示推荐
- 开始菜单、桌面和固定入口会获得额外排序加成
- 对同名快捷方式较多的结果做轻量去重，减少噪声
- 输入现有完整路径或网址可直接启动
- 结果列表使用原生 Windows 文件图标
- 结果列表会显示紧凑的来源目录标签
- 会对常用目录和配置目录的变化做防抖，然后后台自动刷新
- 热键呼出会清空输入，`Ctrl+A` 全选输入，`Ctrl+Shift+Enter` 以管理员运行，`Shift+Enter` 在资源管理器中定位，`Ctrl+C` 复制完整路径
- 主窗口内置 `Run at Startup` 开关，同时托盘菜单保留相同选项作为备用入口
- 托盘菜单提供打开、重建索引、设置、开机自启开关和退出
- 文件夹索引默认来自用户目录以及配置中的 `include` 和 `priority` 目录，以降低后台内存占用

## Settings / 设置
设置文件：`%LOCALAPPDATA%\\QuickLauncherNative\\settings.ini`
- 支持的配置行：
  - `hotkey=Alt+Space` / 设置热键
  - `include=E:\\Tools` / 额外纳入扫描的目录
  - `exclude=E:\\SteamLibrary\\steamapps` / 排除扫描的目录
  - `priority=E:\\Tools` / 排序优先目录
- 修改文件后，可通过托盘菜单 `Reload Settings` 重新加载并重建索引。

## 开机自启
可通过托盘菜单或主窗口中的 `Run at Startup` 开关，将启动器加入或移出
